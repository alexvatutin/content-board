# Промпт для Claude Code: Content Planning Board

## Контекст

Создай веб-приложение "Content Board" — доску для ведения контент-плана по аналогии с agile-доской (Trello/Notion board), но адаптированную под маркетинг и SMM. Приложение должно быть single-page application с локальным хранением данных (localStorage + возможность экспорта/импорта JSON).

## Стек

- **Frontend:** React + TypeScript + Vite
- **Стилизация:** Tailwind CSS
- **Drag & Drop:** @dnd-kit/core + @dnd-kit/sortable
- **Иконки:** lucide-react
- **Хранение данных:** localStorage (с JSON export/import для бэкапа)
- **Дополнительно:** date-fns для работы с датами

## Структура доски

### Главный экран — Calendar Board View

Доска отображает **горизонтальную timeline-ленту по дням** (по умолчанию — текущая неделя, с возможностью переключения на месяц и произвольный диапазон).

- Каждый день — это **колонка**
- Внутри колонки — **карточки постов**, сгруппированные вертикально
- Карточки можно **перетаскивать между днями** (drag & drop) для переноса даты публикации
- Над доской — **навигация по неделям/месяцам** (стрелки влево-вправо, кнопка "Сегодня")
- Фильтры над доской: по соцсети, по статусу, по тегу/рубрике

### Карточка поста (Post Card) — превью на доске

Компактная карточка, которая показывает:

- **Иконка соцсети** (цветовая маркировка) — Instagram, Telegram, YouTube, TikTok, LinkedIn, Facebook, Twitter/X, VK, другое
- **Время публикации** (например, "14:00")
- **Заголовок / первая строка поста** (обрезанный до 60 символов)
- **Статус** — цветовой индикатор:
  - 🔘 `Идея` (серый)
  - ✏️ `Черновик` (жёлтый)
  - 👀 `На согласовании` (оранжевый)
  - ✅ `Готов к публикации` (синий)
  - 🚀 `Опубликован` (зелёный)
  - ❌ `Отменён` (красный)
- **Тег/рубрика** — маленький бейдж (например: "Продукт", "Lifestyle", "Промо", "Образовательный")

### Модальное окно поста (Post Detail Modal)

При клике на карточку открывается модалка с полной информацией и редактированием:

#### Основные поля:
- **Заголовок поста** (text input)
- **Текст поста** (textarea, markdown-friendly, с возможностью скопировать)
- **Соцсеть** (select/dropdown с иконками): Instagram, Telegram, YouTube, TikTok, LinkedIn, Facebook, Twitter/X, VK, Другое
- **Формат контента** (select): Пост, Stories/Reels, Видео, Carousel, Article, Poll, другое
- **Дата публикации** (date picker)
- **Время публикации** (time picker)
- **Статус** (select из списка выше)
- **Тег/рубрика** (input с автокомплитом по ранее использованным тегам, можно несколько)

#### Блок "После публикации":
- **Ссылка на пост** (URL input) — появляется поле, когда статус = "Опубликован"
- **Метрики / охваты** — набор числовых полей:
  - Охват (Reach)
  - Показы (Impressions)
  - Лайки (Likes)
  - Комментарии (Comments)
  - Репосты / Шеры (Shares)
  - Сохранения (Saves)
  - Клики по ссылке (Link clicks)
  - Подписки (Follows gained)
- Возможность добавить **произвольную метрику** (название + значение)

#### Дополнительно:
- **Заметки** (textarea) — внутренние комметрии, не для публикации
- **Кнопка "Дублировать пост"** — создаёт копию с тем же текстом, но без даты и статуса (для кросс-постинга)
- **Кнопка "Удалить"** с подтверждением

## Дополнительный функционал

### 1. Альтернативный вид — List View
Табличный вид всех постов с сортировкой по дате, фильтрацией по соцсети/статусу. Колонки: Дата, Время, Соцсеть, Заголовок, Статус, Охват, Ссылка.

### 2. Быстрое создание
Кнопка "+" на каждой колонке дня для быстрого создания поста — открывает модалку с предзаполненной датой.

### 3. Цветовое кодирование
Каждая соцсеть имеет свой цвет (Instagram — градиент розово-оранжевый, Telegram — голубой, YouTube — красный, TikTok — чёрный, и т.д.). Карточки имеют тонкую цветную полоску слева или сверху.

### 4. Data persistence
- Автосохранение в localStorage при каждом изменении
- Кнопка "Экспорт" — скачивает весь контент-план как JSON
- Кнопка "Импорт" — загружает JSON и мержит/заменяет данные
- При первом запуске — показать пустую доску с подсказкой "Создайте первый пост"

### 5. Статистика (простой дашборд)
Отдельная вкладка/секция "Аналитика":
- Количество постов по соцсетям (bar chart или просто числа)
- Количество постов по статусам
- Средний охват по соцсетям (если данные заполнены)
- Календарная тепловая карта активности (по количеству постов в день)

## UI/UX требования

- **Тёмная тема** по умолчанию, с возможностью переключения на светлую
- **Минималистичный дизайн**, чистые линии, без визуального шума
- **Адаптивность**: на мобильном — колонки скроллятся горизонтально, карточки читаемы
- **Быстрая навигация**: горячие клавиши (N — новый пост, ← → — переключение недель, Esc — закрыть модалку)
- **Smooth animations**: плавные переходы при drag & drop, открытии модалок
- Шрифт — Inter или системный sans-serif

## Модель данных (TypeScript types)

```typescript
type SocialPlatform = 
  | 'instagram' | 'telegram' | 'youtube' | 'tiktok' 
  | 'linkedin' | 'facebook' | 'twitter' | 'vk' | 'other';

type PostStatus = 
  | 'idea' | 'draft' | 'review' | 'ready' | 'published' | 'cancelled';

type ContentFormat = 
  | 'post' | 'stories' | 'video' | 'carousel' | 'article' | 'poll' | 'other';

interface PostMetrics {
  reach?: number;
  impressions?: number;
  likes?: number;
  comments?: number;
  shares?: number;
  saves?: number;
  linkClicks?: number;
  followsGained?: number;
  custom?: Array<{ name: string; value: number }>;
}

interface Post {
  id: string;                    // uuid
  title: string;
  content: string;               // текст поста
  platform: SocialPlatform;
  format: ContentFormat;
  scheduledDate: string;          // ISO date
  scheduledTime: string;          // HH:mm
  status: PostStatus;
  tags: string[];
  publishedUrl?: string;
  metrics?: PostMetrics;
  notes?: string;
  createdAt: string;
  updatedAt: string;
}
```

## Важные детали реализации

1. **Drag & drop** должен обновлять `scheduledDate` поста при перемещении между колонками
2. **Фильтры** работают мгновенно (client-side), без перезагрузки
3. **Модалка** закрывается по Esc и клику вне её, изменения сохраняются при закрытии
4. **Время** отображается в 24-часовом формате
5. **Посты внутри дня** сортируются по времени (ранние сверху)
6. **Пустые дни** показывают placeholder "Нет постов" с кнопкой "+"
7. При экспорте JSON — имя файла включает дату (content-plan-2026-04-05.json)
8. Все тексты интерфейса — на **русском языке**

## Запуск

После создания проекта, убедись что:
- `npm install` проходит без ошибок
- `npm run dev` запускает dev-сервер
- Приложение открывается и работает в браузере
- Drag & drop работает корректно
- Данные сохраняются в localStorage между перезагрузками
